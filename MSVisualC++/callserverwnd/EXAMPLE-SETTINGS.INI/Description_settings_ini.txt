Возможные настройки:
MAX_CLIENT_CONNECTIONS - максимальное число одновременных подключений к серверу сообщений

Программа может работать в каждом режиме индивидуально.
Режим обработки очереди звонков.
Режим обработки очереди сообщений - для синхронизации рабочих мест диспетчеров.
Режим определения номеров с RS232 устройств.
Режим определения номеров с ISDN устройств. ISDN устройство должно иметь драйвер CAPI, 
т.е. где-то должна быть библиотека capi32.dll которая загружается динамически потоком с настройками из isdn.bri.terminal.ХХХ.УУУ параметра.
где-то это значит в пути загрузки библиотек. для виндовс это стандартный путь - см. у РУссиновича путь поиска библиотек.

server.run - запускать сервер обработки сообщений - 1, не запускать сервер - 0

server.port - на каком порту принимать входящие подключения

client.run - запускать в режиме клиента 1, иначе 0. 
Нужен для ISDN. 
Режим клиента - возможность передавать определённые определителем номера на программу запущенную на другом ПК в режиме сервера.

client.connect.host - IP компьютера запущенного в режиме сервера. Нужен для ISDN.

client.connect.port - порт компьютера запущенного в режиме сервера. Нужен для ISDN.

client.prioritet - приоритет клиента, нужен для обработки очереди сообщений. Нужен для ISDN.

serial.terminal.amount - количество запускаемых потоков которые работают с RS323 портами. 
Если равно 0 то все настройки serial.terminal не обрабатываются.

serial.terminal.server.host - IP компьютера на который передаются сообщения с определителя.

serial.terminal.server.port - порт компьютера на который передаются сообщения с определителя.

serial.terminal.number.Х - настройки для конкретного потока определения номера с конкрентого RS232 порта.
Х порядковый номер потока, всегда меньше или равно serial.terminal.amount
Х нужен чтобы различать потоки работающие с RS232 портами т.к. может быть несколько устройств подключённых к одному компьютеру.
Если serial.terminal.amount указан 1 то оработаются только настройки для serial.terminal.number.1
Если serial.terminal.amount указан 2 то оработаются только настройки для serial.terminal.number.1 и serial.terminal.number.2
и так далее.

serial.terminal.number.Х.portname - номер RS232 порта к которому подключено устройство (телефон). 

serial.terminal.number.Х.telef.model - модель телефона или устройства.
Работает с телефонами Нокия, Сониэрикссон, Сименс. Конкретную модель надо проверять.

serial.terminal.number.Х.period - время между сигналами RING. 
Измеряется в миллионах тактов процессорного таймера установленного на материнской плате.
Присутствует на всех компьютерах.
Подбирается вручную и для одного и того же телефона может отличаться на разных компьютерах.

serial.terminal.number.Х.comment - комментарий. передаётся в сообщении.
Служит для идентификации линии. Обычно пишут название телефона.

isdn.bri.terminal.amount - количество запускаемых потоков которые работают ISDN устройствами если их несколько, 
в настоящий момент работа с несколькими ISDN устройствами на одном ПК не поддерживается, параметр введён на будующее.
Если установить 0 то поток для ISDN запускаться не будет.

isdn.bri.terminal.server.host - IP компьютера на который передаются сообщения с определителя.
isdn.bri.terminal.server.port - порт компьютера на который передаются сообщения с определителя.

Примеры настройки:
1) settings-example1.ini - обслуживание очереди сообщений от диспетчерской программы, обмен сообщениями между программами диспетчерами.
2) settings-example2.ini - работа по примеру 1, определение номеров от ISDN и двух сотовых телефонов по RS232 порту, накопление очереди определённых определителем номеров и раздача их диспетчерским программам в соответствии с их приоритетом.
3) settings-example3.ini - определиние номеров только сотового телефона и передача их на хост обслуживающий очередь сообщений от диспетчерских программ.
